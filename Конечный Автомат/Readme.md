# Конечный автомат (Finite State Machine, FSM)

## Что такое конечный автомат

Конечный автомат — это математическая модель системы, которая в любой момент времени находится ровно в **одном** из заранее определённых состояний. При получении входного события система переходит в другое состояние и/или выполняет действие.

## Формальное определение

FSM описывается пятью компонентами:

| Обозначение | Название | Описание |
|-------------|----------|----------|
| **S** | Состояния | Конечное множество всех возможных состояний |
| **Σ** | Алфавит событий | Множество входных сигналов/событий |
| **δ** | Функция переходов | `δ(состояние, событие) → новое состояние` |
| **s₀** | Начальное состояние | Состояние при старте системы |
| **F** | Конечные состояния | Опциональное множество финальных состояний |


## Виды автоматов

### По детерминированности
- **Детерминированный (DFA)** — из одного состояния по одному событию всегда ровно один переход
- **Недетерминированный (NFA)** — по одному событию возможно несколько переходов

### По типу выхода
- **Автомат Мура (Moore)** — выход зависит только от текущего **состояния**
- **Автомат Милли (Mealy)** — выход зависит от **состояния и события**

### Состояния
- `Idle` — ожидание выбора товара
- `WaitingForPayment` — ожидание оплаты
- `Dispensing` — выдача товара

### Таблица переходов

| Состояние | Событие | Условие | Следующее состояние |
|-----------|---------|---------|-------------------|
| Idle | SelectProduct | товар есть | WaitingForPayment |
| Idle | SelectProduct | товара нет | Idle |
| WaitingForPayment | InsertCoin | денег достаточно | Dispensing |
| WaitingForPayment | InsertCoin | денег мало | WaitingForPayment |
| WaitingForPayment | Cancel | — | Idle |
| Dispensing | DispenseProduct | — | Idle |

## Где применяется

- Парсеры и валидаторы (email, номер телефона)
- UI-компоненты (idle → loading → success/error)
- Игровые NPC (патруль → преследование → атака → побег)
- Сетевые протоколы (TCP: CLOSED → LISTEN → ESTABLISHED)
- Бизнес-процессы (заказ: создан → оплачен → отправлен → доставлен)
